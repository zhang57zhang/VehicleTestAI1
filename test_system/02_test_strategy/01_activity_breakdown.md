## 2. 叶子节点详细定义

### 2.1 测试范围定义子活动叶子节点

#### ACT-TS-1-1-001 识别被测系统范围

| 属性 | 内容 |
|------|------|
| **活动编号** | ACT-TS-1-1-001 |
| **活动名称** | 识别被测系统范围 |
| **所属层级** | 4-叶子节点 |
| **英文名称** | Identify System Under Test Scope |
| **活动描述** | 系统性地识别被测系统（SUT）的范围和边界，明确测试活动将覆盖的控制器及其功能。对于车载运动域/底盘域控制器，需要确定被测控制器（如VCU、ESP、EPS等）的功能范围、通信接口、运行环境等。被测系统范围的准确定义是制定有效测试策略的基础，决定了测试的完整性和效率。 |
| **入口条件** | 1. 系统架构文档已获取<br>2. 需求分析报告已完成<br>3. 功能分解结构已建立 |
| **出口条件** | 1. 完成被测系统范围定义<br>2. 建立系统边界文档<br>3. 识别系统与外部交互 |
| **入口文档** | 1. 系统架构设计文档<br>2. 需求分析报告<br>3. 功能分解结构图 |
| **出口文档** | 1. 被测系统范围定义<br>2. 系统边界说明<br>3. 外部交互清单 |
| **前序依赖** | ACT-RA-3-1-001（建立功能分解结构） |
| **责任角色** | 测试策略工程师，系统工程师 |
| **工时估算** | 8小时/人日 |
