## 2. 叶子节点详细定义

### 2.1 需求获取子活动叶子节点

#### ACT-RA-1-1-001 识别法规标准需求

| 属性 | 内容 |
|------|------|
| **活动编号** | ACT-RA-1-1-001 |
| **活动名称** | 识别法规标准需求 |
| **所属层级** | 4-叶子节点 |
| **英文名称** | Identify Regulatory Standard Requirements |
| **活动描述** | 系统性地识别与车载运动域/底盘域控制器相关的法规标准要求，包括国际标准、国家标准、行业标准以及企业标准。对于底盘域控制器，需要重点关注功能安全标准（ISO 26262）、排放标准（Euro 6/China 6）、安全标准（UN R13H等）以及电磁兼容标准（ISO 11451/11452）。识别结果将作为需求分析的输入，确保测试活动符合法规要求。 |
| **入口条件** | 1. 项目启动文件已批准<br>2. 相关法规标准清单已获取<br>3. 标准查询渠道已建立 |
| **出口条件** | 1. 完成法规标准需求识别报告<br>2. 建立法规标准清单<br>3. 完成法规条款与需求映射表 |
| **入口文档** | 1. 项目启动计划书<br>2. 适用法规标准清单 |
| **出口文档** | 1. 法规标准需求识别报告<br>2. 法规标准清单（按产品族/市场区域分类）<br>3. 法规条款与需求映射表 |
| **前序依赖** | 无 |
| **责任角色** | 需求分析师，功能安全工程师 |
| **工时估算** | 8小时/人日 |

#### ACT-RA-1-1-002 识别客户需求

| 属性 | 内容 |
|------|------|
| **活动编号** | ACT-RA-1-1-002 |
| **活动名称** | 识别客户需求 |
| **所属层级** | 4-叶子节点 |
| **英文名称** | Identify Customer Requirements |
| **活动描述** | 从客户规范、技术协议、质量要求等文档中提取和整理客户对底盘域控制器的具体需求。客户需求通常以自然语言描述，需要经过分析转化才能形成可测试的需求规格。客户需求包括功能需求、性能指标、接口要求、可靠性要求、交付要求等内容。对于车载控制器，还需要关注OEM厂商的特殊要求，如特定的诊断协议、标定参数范围等。 |
| **入口条件** | 1. 客户合同/协议已签订<br>2. 客户需求文档已提供<br>3. 需求分析师已分配 |
| **出口条件** | 1. 完成客户需求清单<br>2. 完成需求初步分类（功能/性能/接口/安全）<br>3. 识别需求冲突或歧义 |
| **入口文档** | 1. 客户技术协议<br>2. 客户质量要求<br>3. 接口矩阵表 |
| **出口文档** | 1. 客户需求清单<br>2. 需求分类表<br>3. 需求疑问清单 |
| **前序依赖** | ACT-RA-1-1-001（识别法规标准需求） |
| **责任角色** | 需求分析师 |
| **工时估算** | 16小时/人日 |

| **入口条件** | 1. SRS文档已起草或更新<br>2. 文档评审流程已完成<br>3. 文档配置管理已建立 |
