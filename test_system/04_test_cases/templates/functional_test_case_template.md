## 1 模板说明

本模板用于编写车载运动域/底盘域控制器的功能测试用例。功能测试用例验证被测系统的功能需求是否正确实现，包括正常功能测试、异常功能测试和边界条件测试。测试用例编写应遵循测试用例编写规范，确保用例的完整性、正确性和可追溯性。

## 3 填写示例

### 3.1 示例测试用例

| 字段 | 内容 |
|------|------|
| 用例编号 | TC-FUNC-VCU-001 |
| 用例名称 | 整车控制器电源管理功能测试 |
| 用例类型 | 功能测试 |
| 所属模块 | 整车控制器VCU |
| 优先级 | 高 |
| 关联需求 | REQ-VCU-PWR-001, REQ-VCU-PWR-002 |
| 关联场景 | SCN-VCU-PWR-001 |
| 编写人 | 张三 |
| 编写日期 | 2026-04-01 |

**测试目的：**
验证VCU在各种电源模式下的上下电时序和控制逻辑是否符合设计要求，确保电源管理功能正确实现。

**前置条件：**
1. VCU硬件已上电完成，自检通过
2. VCU与整车CAN网络通信正常
3. 12V电源供电稳定，电压在10.8V~16V范围内
4. 诊断仪已连接，可监控VCU内部状态

**测试输入：**

| 序号 | 输入名称 | 输入值 | 输入来源 | 准备要求 |
|------|----------|--------|----------|----------|
| 1 | 点火开关信号 | ON/OFF/ACC | 钥匙信号模拟器 | 按测试步骤切换状态 |
| 2 | 12V电源电压 | 12V | 电源模拟器 | 设置稳定电压值 |
| 3 | CAN报文 | 车速信号 | CAN总线仿真器 | 发送有效车速数据 |

**操作步骤：**

| 步骤 | 操作描述 | 预期中间结果 |
|------|----------|--------------|
| 1 | 将点火开关从OFF切换至ON | VCU检测到KL15上电信号 |
| 2 | 等待VCU完成初始化（约500ms） | VCU内部状态变为“运行就绪” |
| 3 | 通过诊断仪读取VCU电源模式 | 电源模式显示为“KL15_on” |
| 4 | 观察VCU输出继电器状态 | 主继电器闭合，12V输出正常 |
| 5 | 将点火开关从ON切换至OFF | VCU检测到KL15下电信号 |
| 6 | 等待VCU进入休眠（约5s） | VCU内部状态变为“深度休眠” |
| 7 | 通过诊断仪读取VCU电源模式 | 电源模式显示为“KL15_off” |

**预期结果：**
1. VCU能够正确检测点火开关状态变化，状态检测延迟小于10ms
2. VCU上电后能够在500ms内完成初始化，进入运行就绪状态
3. VCU上电后主继电器闭合时间为200ms~400ms
4. VCU下电后能够在5s内进入深度休眠状态
5. VCU电源模式状态与点火开关状态一致

**后置条件：**
1. 将点火开关置于OFF位置
2. 断开诊断仪连接
3. VCU进入深度休眠状态

### 3.2 填写规范说明

**用例编号填写规范：**
- 格式：TC-FUNC-[模块英文缩写]-[三位序号]
- 模块英文缩写：VCU-整车控制器、ESP-车身稳定系统、EPS-电子助力转向、EPB-电子驻车制动
- 序号：001~999，连续编号

**用例名称填写规范：**
- 简洁明确，不超过50个字符
- 采用“对象+动作+预期结果”的命名方式
- 避免使用模糊或笼统的名称

**测试目的填写规范：**
- 明确具体，说明此用例要验证什么
- 与关联需求对应，便于追溯
- 避免过于简略或过于详细

**前置条件填写规范：**
- 明确具体，列出所有必要的前置条件
- 包括环境状态、数据状态和配置状态
- 前置条件应可验证

**测试输入填写规范：**
- 详细列出所有测试输入
- 明确输入值、输入来源和准备要求
- 输入数据应具有代表性和可重复性

**操作步骤填写规范：**
- 每一步应清晰准确，避免歧义
- 明确操作主体（测试人员/被测系统/测试工具）
- 预期中间结果应与操作步骤对应

**预期结果填写规范：**
- 明确可验证，避免模糊的描述
- 包括功能结果、状态变化和输出数据
- 结果应可通过观察或测量验证
