## 2. 叶子节点详细定义

### 2.1 测试架构设计子活动叶子节点

#### ACT-TD-1-1-001 设计测试层次结构

| 属性 | 内容 |
|------|------|
| **活动编号** | ACT-TD-1-1-001 |
| **活动名称** | 设计测试层次结构 |
| **所属层级** | 4-叶子节点 |
| **英文名称** | Design Test Hierarchy Structure |
| **活动描述** | 设计测试用例的层次化组织结构，建立从测试套件到测试组再到单个测试用例的层级关系。测试层次结构的设计应当与被测系统的结构保持一致，便于测试用例的管理和维护。对于底盘域控制器，测试层次结构通常包括系统测试层、集成测试层、组件测试层等。清晰的测试层次结构有助于测试执行的管理和测试报告的组织。 |
| **入口条件** | 1. 系统架构文档已获取<br>2. 测试策略已批准<br>3. 测试范围已定义 |
| **出口条件** | 1. 完成测试层次结构设计<br>2. 建立测试套件清单<br>3. 建立测试分组定义 |
| **入口文档** | 1. 系统架构设计文档<br>2. 测试策略文档<br>3. 测试范围定义 |
| **出口文档** | 1. 测试层次结构图<br>2. 测试套件清单<br>3. 测试分组定义 |
| **前序依赖** | ACT-TS-1-1-001（识别被测系统范围），ACT-TS-1-2-001（定义组件测试级别） |
| **责任角色** | 测试设计工程师，测试架构师 |
| **工时估算** | 8小时/人日 |

#### ACT-TD-1-1-002 定义测试分组策略

| 属性 | 内容 |
|------|------|
| **活动编号** | ACT-TD-1-1-002 |

| **所属层级** | 4-叶子节点 |
| **英文名称** | Define Test Grouping Strategy |
| **活动描述** | 定义测试用例分组的策略和规则，包括按功能分组、按测试类型分组、按执行环境分组等策略。测试分组策略应当考虑测试执行效率、测试报告组织和测试结果分析的需要。对于底盘域控制器，测试分组策略通常包括按控制功能分组（动力管理、底盘稳定、转向控制等）、按测试类型分组（功能测试、性能测试、安全测试等）和按测试环境分组（MIL、SIL、HIL、实车等）。 |
| **入口条件** | 1. 测试层次结构已设计<br>2. 测试类型已规划<br>3. 测试用例已开始设计 |
| **出口条件** | 1. 完成测试分组策略定义<br>2. 建立分组命名规范<br>3. 定义分组执行规则 |
| **入口文档** | 1. 测试层次结构图<br>2. 测试类型规划<br>3. 测试用例清单（草稿） |

| **前序依赖** | ACT-TD-1-1-001（设计测试层次结构） |
| **责任角色** | 测试设计工程师 |
| **工时估算** | 6小时/人日 |
